# API Практикум — Система тестов

## Обзор

На странице `/glearning` расположен курс из 16 видеоуроков. После ввода API-ключа
студент получает доступ к практическим тестам — чекбоксам с багами, которые нужно
найти в реальном API (`practicum.eddytester.com`).

Тесты загружаются динамически из бэкенда (`v0-test-api`, порт 3000).
Прогресс сохраняется в PostgreSQL, привязан к API-ключу студента.

## Механика работы

1. **Вход**: студент вводит API-ключ в оверлей (`#gk-overlay`)
2. **Проверка**: ключ проходит через `/check-key` (прокси → `key-check` на порту 3456)
3. **Загрузка тестов**: после успешной проверки клиент запрашивает `/get-progress?key=...`
4. **Рендер**: JS динамически создаёт чекбоксы с багами в DOM урока (l9, l12–l15)
5. **Выбор**: студент отмечает баги, которые нашёл в API, и нажимает «Проверить»
6. **Сабмит**: `POST /save-progress` отправляет выбранные баги на сервер
7. **Результат**: сервер сверяет с эталоном (`LESSONS_CONFIG`), возвращает правильные/неправильные
8. **Прогресс-бар**: обновляется в сайдбаре (пройдено/N)
9. **Повтор**: если не все баги найдены — кнопка «Попробовать ещё раз»
10. **Завершение**: когда все 5 уроков пройдены — курс считается завершённым

## Техническая реализация

### Фронтенд

- **Файл:** `page101922726.html` (весь HTML + CSS + JS инлайн)
- **LESSON_MAP:** маппинг id урока → lesson_id бэкенда:
  ```
  l9  → get-users
  l12 → get-user
  l13 → post-users
  l14 → patch-user
  l15 → delete-user
  ```
- **loadProgress(key):** GET `/get-progress?key=...` → рендер чекбоксов
- **submitTest(btn):** POST `/save-progress` → applyResults()
- **localStorage:** API-ключ кэшируется (`gk_key`), прогресс перезагружается при каждом входе
- MutationObserver отслеживает скрытие оверлея, чтобы подгрузить прогресс

### Бэкенд

- **Стек:** Node.js + Express + PostgreSQL
- **Расположение:** `/var/www/v0-test-api/v0-test-api/server.js`
- **Порт:** 3000 (прокси через Apache: `/get-progress` и `/save-progress`)

### API эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/get-progress?key=<api_key>` | Получить список уроков, багов и прогресс |
| `POST` | `/save-progress` | Сохранить результат попытки |

### База данных (PostgreSQL)

- **Таблица:** `lesson_progress`
  - `api_key` — ключ студента
  - `lesson_id` — идентификатор урока
  - `selected_bugs` — выбранные баги (массив)
  - `correct_bugs` — правильные ответы
  - `wrong_bugs` — неверные ответы
  - `completed` — boolean, пройден ли урок
  - `attempts` — количество попыток

### Конфигурация уроков

Правильные ответы для каждого урока заданы в `LESSONS_CONFIG` в
`/var/www/v0-test-api/v0-test-api/src/routes/progress.js`.

Редактировать там же: добавить/убрать баг, изменить correct-массив.
